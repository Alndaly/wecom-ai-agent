# 企业微信(WeCom)操作手册 (V3.2 - 结构模式精简版)

供 ReAct Agent 决策参考。定位以 **page、节点 cls/editable、相对位置、容器结构、流程状态** 为准，避免依赖具体 UI 文字。

## 1. 页面分类与状态判定

| page 值 | 含义 | 关键结构特征 |
|---|---|---|
| `HOME` | 会话列表主页 | 中部纵向列表；列表项为整行容器，含头像、标题/摘要/时间等子节点。 |
| `CHAT` | 单个会话页 | 顶部标题区；中部消息流；底部输入栏/功能栏。 |
| `UNKNOWN` | 非主会话页或弹层 | 常见于附件面板、媒体选择器、预览确认页、搜索页等；需按结构继续判断。 |

**核心原则**
1. 优先使用 `page`，但 `UNKNOWN` 不代表失败；媒体链路中多数中间页为 `UNKNOWN`。
2. 若已在目标 `CHAT`，且顶部标题区/上下文确认是目标会话，直接执行发送流程，不要返回重进。
3. `CHAT` 中是否可输入由 `editable=true` 判定；无 editable 时可能处于语音/非文本模式。
4. 媒体流程中一旦出现附件网格、媒体网格、预览主体、右下确认按钮等结构，应沿流程推进，避免无依据 `back`。

## 2. 进入目标会话

### 2.1 从会话列表进入

当前 `page=HOME` 时：
- 优先点击缓存 locator：`role=chat_target`。
- 目标通常是中部列表中的整行可点击容器，左侧头像，右侧多行信息。
- 不要只点头像或摘要小节点；优先点整行容器中心区域。

**成功判定**
- 进入 `page=CHAT`；
- 顶部约 15% 区域存在稳定标题/会话信息；
- 当前会话与目标匹配，或任务上下文已确认。

### 2.2 已在会话页

当前 `page=CHAT` 且顶部会话与目标一致：
- 发送文本：直接找 `message_input`；
- 发送媒体：直接点底部最右 `compose_plus`；
- 不必回到 `HOME` 再点击 `chat_target`。

### 2.3 搜索路径

若 `HOME` 列表不可见或目标项不存在：
1. 点击顶部搜索入口；
2. 在搜索页 editable 输入目标关键词；
3. 点击结果列表中最匹配的整行容器；
4. 验证进入目标 `CHAT`。

## 3. 文本发送流程

### 3.1 底部输入栏结构

`CHAT` 底部通常为一行：

`[最左模式切换图标] [中间输入区域] [辅助图标] [最右附件/更多图标]`

- 文本模式：存在 `editable=true` 的输入节点，通常位于底部中间。
- 非文本模式：无 editable，中间可能是不可编辑控件；需切回文本模式。
- 输入文本后：底栏右侧会出现可点击发送动作节点，位置与输入框同一行、靠右。

### 3.2 标准文本发送

1. **确认会话**
   - 若在目标 `CHAT`：继续；
   - 若在 `HOME`：点 `chat_target`；
   - 其它页：优先回到可识别的 `CHAT/HOME` 后继续。

2. **定位输入框**
   - 若 UI tree 中存在底部 `editable=true`：使用 `role=message_input`。
   - 若 `page=CHAT` 但无 editable：
     - 点击底部同一行最左侧 ImageView/模式切换图标；
     - `wait_ui`；
     - 再查找底部 `editable=true`。
   - 注意：恢复文本模式时不要点中间不可编辑区域，应点最左模式图标。

3. **输入**
   - 对 `role=message_input` 执行 `input_text`。
   - 输入后等待底栏右侧动作节点刷新。

4. **发送**
   - 点击输入栏同一行右侧可点击动作节点：`role=send_button`。
   - 成功后仍在 `CHAT`，输入框清空/恢复占位，或消息流底部新增文本气泡。

## 4. 媒体发送流程

媒体链路：`CHAT` 打开附件面板 → 点媒体入口 → 在媒体网格选真实缩略图 → 预览/确认页右下确认 → 等回 `CHAT`。

### 4.1 打开附件面板

**前提**：当前在目标 `CHAT`。

**操作**
- 点击底部输入栏同一行最右侧 ImageView/功能图标：`role=compose_plus`。
- 它位于辅助/表情图标右侧，是最靠右入口。

**避免**
- 不点倒数第二个辅助图标。
- 若打开了非附件子面板，可 `back` 收起或重新定位底栏最右入口。

### 4.2 附件面板中进入媒体选择器

附件面板可能仍显示 `CHAT`，也可能为 `UNKNOWN`。

**结构信号**
- 屏幕下半部或底部出现多列功能网格；
- 每格为“上方图标 + 下方短标签”的可点击容器；
- 媒体库入口通常在第一屏靠前位置，常见为左上或前几个格子；
- 入口图标具有图片/相册类视觉特征，但不要依赖文字。

**操作**
- 点击网格靠前的本地媒体库入口：`role=media_picker_entry`。
- 若点错其它功能格，返回上一层或回到 `CHAT` 后重新从 `compose_plus` 开始。

### 4.3 媒体选择器中选择目标媒体

进入后通常 `page=UNKNOWN`，主体为图片/视频网格。

**真实媒体缩略图结构**
- 方形/矩形缩略图容器；
- 子节点多为 ImageView；
- 可能叠加勾选框、播放标记、时长角标；
- 通常没有短文字标签；
- 顶部/首个格子可能是工具型入口，含文本标签或非缩略图布局，应跳过。

**选择规则**
1. 扫描主体媒体网格；
2. 跳过含文本标签的快捷/工具/非媒体入口；
3. 选择第一个真实媒体缩略图容器：`role=gallery_first_item`；
4. 刚保存/刚生成的图片、GIF、视频通常位于最新区域，即跳过工具入口后的第一个真实缩略图；
5. 带时长角标或播放叠层的缩略图可视为视频真实媒体项。

### 4.4 预览/确认页发送

选择缩略图后进入 `UNKNOWN` 预览/确认页。

**结构信号**
- 主体区域显示大图、视频预览或选中媒体；
- 底部或右下角存在可点击确认动作节点；
- 该节点通常为右下矩形/文本按钮；
- 左上返回/关闭、底部编辑/选项/计数等为辅助节点，不用于发送。

**操作**
- 点击右下确认动作：`role=gallery_send_button`。
- 点击后优先 `wait_ui`，等待页面回到目标 `CHAT`。

### 4.5 发送后的等待与恢复

点击 `gallery_send_button` 后：
- 若仍停留 `UNKNOWN`，先 `wait_ui`，不要立即 `back`；
- 页面可能短暂停留预览、选择器或 loading 状态；
- 无明确失败前，`back` 可能取消发送或打断返回链路。

**成功信号**
- 从媒体选择/预览相关页回到 `CHAT`；
- 顶部会话仍为发送前目标；
- 消息流底部出现媒体气泡、上传占位或发送中状态；
- 输入栏恢复正常。

若误返回到 `HOME`：
- 说明流程已脱离当前会话；
- 重新点击 `chat_target` 进入目标会话；
- 若此前已点过 `gallery_send_button`，进入后先观察消息流底部，不要重复发送，除非确认未出现媒体气泡/发送中项。

## 5. 常用角色定位摘要

| 角色 | 使用场景 | 结构/位置定位 |
|---|---|---|
| `chat_target` | 打开目标会话 | `HOME` 中部会话列表整行容器；含头像与多行会话信息；优先缓存 locator。 |
| `message_input` | 输入文本 | `CHAT` 底部中间 `editable=true` 节点；若无，先点底栏最左模式切换图标。 |
| `send_button` | 发送文本 | 输入后底部输入栏同一行右侧出现的可点击动作节点。 |
| `compose_plus` | 打开附件面板 | `CHAT` 底部输入栏同一行最右侧 ImageView/功能图标。 |
| `media_picker_entry` | 进入媒体库 | 附件面板网格第一屏靠前的“图标 + 标签”媒体类入口格。 |
| `gallery_first_item` | 选择最新媒体 | 媒体网格中跳过文字型工具入口后的第一个真实缩略图容器。 |
| `gallery_send_button` | 确认发送媒体 | 预览/确认页右下角可点击确认动作节点。 |

## 6. 成功判定与恢复策略

### 6.1 文本发送成功

满足任一结构信号：
- 点击 `send_button` 后仍在目标 `CHAT`，输入框清空/恢复占位；
- 消息流底部新增文本气泡；
- 右侧发送动作消失或恢复为附件/更多图标。

### 6.2 媒体发送成功

优先用组合条件：
1. 已点击 `gallery_send_button`；
2. 页面从 `UNKNOWN` 媒体页回到 `CHAT`；
3. 顶部会话与发送前一致；
4. 消息流底部出现媒体气泡、上传占位或发送中项。

### 6.3 常见问题

| 现象 | 结构原因 | 推荐处理 |
|---|---|---|
| `CHAT` 无输入框 | 底部非文本模式 | 点底部最左模式切换图标，等待 editable 出现。 |
| 点开辅助/表情面板 | 点到最右入口左侧图标 | `back` 收起，重新点底栏最右 `compose_plus`。 |
| 附件面板入口点错 | 点到其它功能格 | 返回上一层或回 `CHAT`，再点靠前媒体入口。 |
| 媒体网格点到工具项 | 未跳过含文本标签入口 | 返回选择器，点第一个纯缩略图/带叠层真实媒体项。 |
| 预览确认后仍 `UNKNOWN` | 发送处理中或 UI 未刷新 | 先 `wait_ui`；不要立即 `back`。 |
| 回到 `HOME` | 返回链路被打断或误按返回 | 重新进目标会话；若已确认发送，先观察底部消息。 |
| 已在目标聊天页 | 无需导航 | 直接执行文本或媒体发送。 |

## 7. 推荐执行模板

### 7.1 打开会话
1. 若当前为目标 `CHAT`：完成。
2. 若当前为 `HOME`：点击 `role=chat_target`。
3. 若目标项不可见：走搜索路径。
4. 验证顶部标题区/会话上下文。

### 7.2 发送文本
1. 确认在目标 `CHAT`；否则先打开会话。
2. 查找底部 `editable=true` 的 `message_input`。
3. 若无 editable，点底部最左模式切换图标并等待。
4. `input_text(role=message_input)`。
5. 点击右侧 `role=send_button`。
6. 观察输入栏清空或消息流新增。

### 7.3 发送图片/GIF/视频
1. 确认在目标 `CHAT`。
2. 点击底部最右 `role=compose_plus`。
3. 在附件网格点击靠前媒体库入口 `role=media_picker_entry`。
4. 在媒体网格跳过文字型工具入口，点击第一个真实缩略图 `role=gallery_first_item`。
5. 在预览/确认页点击右下 `role=gallery_send_button`。
6. `wait_ui` 等回目标 `CHAT`，确认媒体气泡/发送中项出现。
