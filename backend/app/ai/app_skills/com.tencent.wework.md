# 企业微信(WeCom)操作手册 (V3.1 - 轨迹增强版)

供 ReAct Agent 每次决策参考。内容强调 **page、节点结构、位置、角色与流程状态**，避免依赖具体 UI 文案。

## 1. 页面分类与状态判定

| page 值 | 含义 | 关键结构特征 |
|---|---|---|
| `HOME` | 主页/会话列表页 | 中部为纵向会话列表；每个列表项包含头像/标题/摘要/时间等子节点。 |
| `CHAT` | 单个会话页 | 顶部为会话标题区；中部为消息流；底部为输入/功能栏。 |
| `UNKNOWN` | 非主会话页 | 可能是附件面板、媒体选择器、预览确认页、搜索页、设置页等；需结合节点结构继续判断。 |

**核心原则：**
1. **优先看 `page` 字段**。`page=CHAT` 才按聊天页流程处理；`page=UNKNOWN` 不等于失败，媒体流程中很常见。
2. **已在目标聊天页时可跳过导航**：若当前 `page=CHAT` 且顶部会话标题区与目标一致，可直接执行输入或附件流程。
3. **输入模式用结构判定**：`page=CHAT` 但无任何 `editable=true` 节点时，认为底部处于非文本输入模式。
4. **媒体流程连续性优先**：只要 UI 出现附件面板网格、相册网格、媒体预览、底部/右下确认按钮等结构，应继续媒体流程，不要随意 `back`。

## 2. 聊天页核心流程 (`page=CHAT`)

### 2.1 进入目标会话

**A. 直接路径**
- 当前 `page=HOME` 时，在会话列表中点击目标列表项。
- 优先使用缓存 locator：`role=chat_target`。
- 会话列表项通常是中部纵向列表中的一整行容器，含头像节点与标题/摘要文本节点。

**B. 已在会话页**
- 若当前已是 `page=CHAT`，且顶部标题区与目标一致，无需返回列表重进。
- 可直接进入文本发送或媒体发送步骤。

**C. 搜索路径**
- 当 `HOME` 列表不可见或找不到目标项时，点击顶部搜索入口。
- 在搜索输入框输入目标关键词，再点击结果列表中最匹配的整行容器。
- 成功后应进入 `page=CHAT`，顶部标题区与目标一致。

**成功判定**
- `page=CHAT`；
- 顶部约 15% 区域存在稳定标题/会话信息；
- 标题区与目标一致，或本轮任务上下文已确认当前会话为目标。

## 3. 文本发送流程

### 3.1 底部输入栏结构

聊天页底部一行通常呈现：

`[左侧模式切换图标] [中间输入区域] [表情/辅助图标] [最右附件/更多图标]`

不同模式下中间区域结构不同：
- **文本模式**：存在 `editable=true` 的 EditText/输入节点。
- **非文本模式**：无 `editable=true` 节点，底部可能显示语音/按住类控件或其它非文本状态。

### 3.2 发送文本标准步骤

1. **确认会话**
   - 若不在目标 `CHAT`，先执行 2.1。
   - 若已在目标 `CHAT`，直接继续。

2. **确认文本模式**
   - 若 UI tree 中存在 `editable=true` 节点，使用 `role=message_input`。
   - 若 `page=CHAT` 但不存在 `editable=true`：
     - 点击底部输入栏最左侧的模式切换 ImageView/图标；
     - 等待 UI 刷新；
     - 再查找 `editable=true` 输入节点。
   - 注意：此时的点击目标虽然可使用 `role=message_input` 恢复文本模式，但实际位置应是底栏最左侧模式切换图标，而不是中间不可编辑区域。

3. **输入文本**
   - 对 `role=message_input` 执行 `input_text`。
   - 输入后底部右侧通常会出现可发送的按钮节点，替代或靠近原最右功能图标。

4. **点击发送**
   - 点击输入栏右侧、与输入内容同一行的可发送按钮：`role=send_button`。
   - 成功后仍在 `CHAT`，输入框内容清空或恢复占位状态，消息流底部出现新消息。

## 4. 媒体发送流程

媒体发送通常为四段：聊天页打开附件面板 → 进入媒体选择器 → 选择真实媒体缩略图 → 预览页确认。

### 4.1 打开附件面板

**前提**：当前在目标 `CHAT`。

**操作**
- 点击底部输入栏同一行最右侧的 ImageView/功能图标：`role=compose_plus`。
- 它位于表情/辅助图标右侧，是最靠右的入口。

**避免误点**
- 不要点击输入栏右侧倒数第二个表情/辅助图标。
- 若误入表情/子面板，可先 `back` 或重新点击底栏目标入口恢复，再点击最右附件入口。

### 4.2 进入媒体选择器

附件面板通常仍可能显示为 `CHAT` 或 `UNKNOWN`，底部/下半屏出现功能网格。

**结构信号**
- 面板为多列网格；
- 每个格子通常由上方图标 + 下方短标签组成；
- 媒体选择入口常位于网格靠前位置，通常是第一屏左上或前几个格子；
- 该入口用于进入系统/应用内图片视频选择器。

**操作**
- 点击附件面板中表示本地媒体库的格子：`role=media_picker_entry`。
- 不依赖具体文字，优先依据：网格靠前、图标为图片/相册类、子节点为“图标 + 标签”的入口格。

### 4.3 在相册/图片视频选择器中选择目标媒体

进入后常为 `page=UNKNOWN`，主体是媒体网格。

**识别真实媒体缩略图**
- 真实媒体项通常是方形/矩形缩略图容器；
- 子节点多为 ImageView，可能叠加勾选框、时长角标、播放标记；
- 不应选择带明显文字标签的快捷入口/工具入口；
- 第一屏顶部可能有一个工具类入口，其容器含文本标签，应跳过。

**选择规则**
1. 先扫描媒体网格；
2. 跳过含文字标签的快捷入口、工具入口、非媒体入口；
3. 选择第一个真实媒体缩略图容器，通常是跳过快捷入口后的第一个纯缩略图：`role=gallery_first_item`；
4. 对刚保存/刚生成的图片、GIF、视频，通常选择网格中最新的第一个真实媒体项；
5. 若目标是视频，含时长角标的缩略图也可视为真实媒体项。

### 4.4 预览/确认页发送

选择媒体后通常进入 `page=UNKNOWN` 的预览或确认页。

**结构信号**
- 页面中部/主体显示大图、视频预览或选中媒体；
- 底部或右下角存在确认动作按钮；
- 该按钮通常是右下角矩形/文本按钮，处于可点击状态；
- 顶部可能有返回/关闭，底部可能有编辑、原图、计数等辅助项。

**操作**
- 点击预览页右下角确认发送按钮：`role=gallery_send_button`。
- 不要点击左上返回、关闭、编辑、预览主体或其它辅助控件。

### 4.5 媒体发送后的等待与判定

点击 `gallery_send_button` 后：
- 优先 `wait_ui` 等待页面回到 `CHAT`；
- 短时间停留在 `UNKNOWN`、预览页或 loading 状态时，不要立即 `back`；
- `back` 可能取消发送或回到会话列表，除非确认已偏离流程且无发送动作在进行。

**成功信号**
- 页面从媒体选择/预览相关状态回到 `CHAT`；
- 顶部会话标题区仍为目标会话；
- 输入栏恢复正常，消息流底部出现新媒体气泡，或发送中的媒体气泡可见。

## 5. 常用角色定位摘要

| 角色 | 使用场景 | 结构/位置定位 |
|---|---|---|
| `chat_target` | 从会话列表进入目标会话 | `HOME` 中部列表项整行容器，含头像与会话信息；优先缓存 locator。 |
| `message_input` | 文本输入 | `CHAT` 底部输入栏中的 `editable=true` 节点；若不存在，先点底栏最左模式切换图标。 |
| `send_button` | 发送文本 | 输入文本后，底部输入栏右侧出现的可点击按钮。 |
| `compose_plus` | 打开附件面板 | `CHAT` 底部输入栏同一行最右侧 ImageView/功能图标。 |
| `media_picker_entry` | 进入相册/媒体库 | 附件面板网格靠前的“图标 + 标签”媒体入口格。 |
| `gallery_first_item` | 选择最新媒体 | 媒体网格中第一个真实缩略图；跳过含文字标签的工具/快捷入口。 |
| `gallery_send_button` | 确认发送媒体 | 预览/确认页右下角可点击确认按钮。 |

## 6. 成功判定与恢复策略

### 6.1 文本发送成功

满足以下任一组结构信号即可认为成功：
- 点击 `send_button` 后仍在目标 `CHAT`，输入框清空/恢复占位；
- 消息流底部出现新文本气泡；
- 发送按钮消失或恢复为附件/更多图标。

### 6.2 媒体发送成功

更严格地使用以下条件：
1. 上一步关键动作是点击 `gallery_send_button`；
2. 页面从 `UNKNOWN` 或媒体预览/选择状态回到 `CHAT`；
3. 顶部标题区与发送前会话一致；
4. 消息流底部出现媒体气泡、上传占位或发送中状态。

### 6.3 常见问题处理

| 现象 | 结构原因 | 推荐处理 |
|---|---|---|
| `CHAT` 中找不到输入框 | 底部处于非文本模式 | 点击底部最左模式切换图标，等待出现 `editable=true`。 |
| 打开了表情/辅助面板 | 误点最右入口左侧图标 | `back` 收起或重新定位底部最右 `compose_plus`。 |
| 附件面板里选错入口 | 点到其它功能格 | 返回上一层或回到聊天页，重新选择靠前的媒体库入口格。 |
| 相册中点到工具入口 | 未跳过含文字标签的快捷入口 | 返回选择器，选择第一个纯缩略图/带勾选框真实媒体项。 |
| 预览页点击后仍是 `UNKNOWN` | 发送处理中或 UI 未刷新 | 先 `wait_ui`；不要立即 `back`。 |
| 回到 `HOME` | 可能误按返回或流程被中断 | 重新点击 `chat_target` 进入目标会话，再按任务类型继续。 |
| 当前已在目标聊天页 | 无需重复打开会话 | 直接输入文本或点击 `compose_plus`。 |

## 7. 推荐执行模板

### 7.1 打开会话
1. 若当前为目标 `CHAT`：完成。
2. 若当前为 `HOME`：点击 `role=chat_target`。
3. 若列表无目标：用搜索路径进入。
4. 验证顶部标题区。

### 7.2 发送文本
1. 确认在目标 `CHAT`。
2. 查找 `editable=true` 的 `message_input`。
3. 若无输入框，点击底部最左模式切换图标并等待。
4. `input_text(role=message_input)`。
5. `tap_node(role=send_button)`。
6. 观察输入栏清空或消息流新增。

### 7.3 发送图片/GIF/视频
1. 确认在目标 `CHAT`。
2. 点击底部最右 `role=compose_plus`。
3. 在附件网格中点击媒体库入口 `role=media_picker_entry`。
4. 在媒体网格跳过文字型快捷入口，点击第一个真实缩略图 `role=gallery_first_item`。
5. 在预览页点击右下确认按钮 `role=gallery_send_button`。
6. `wait_ui` 等待回到目标 `CHAT`，确认媒体气泡出现。
