# 企业微信(WeCom)操作手册 (V3.0 - 结构优化版)

供 ReAct Agent 在每次决策时参考。内容强调**结构、位置和流程状态**，而非具体文字。

## 1. 页面分类与状态判定

| page 值 | 含义 | 关键特征 |
|---------|------|------|
| `HOME` | 主页/会话列表页 | 包含会话列表，用于选择目标联系人。 |
| `CHAT` | 单个会话聊天页 | 顶部包含联系人信息，底部包含输入栏。 |
| `UNKNOWN` | 其它/非聊天页 | 相册选择器 (picker)、图片预览、发送确认页、设置页等。 |

> **核心判定原则**:
> 1. **状态优先**: 任何状态的判定，**优先看 `page` 字段**。
> 2. **输入框判定**: 如果 `page=CHAT`，但 UI tree 中找不到任何 `editable=true` 的节点，则判定为**非文本输入模式**。
> 3. **媒体流程**: `page=UNKNOWN` 不一定是错误状态。只要 UI tree 中出现媒体发送相关的控件（如大图预览、勾选框、发送按钮），应继续执行媒体发送流程。

## 2. 聊天页 (page=CHAT) 核心流程

### 2.1 目标会话进入 (Role-Based Navigation)
*   **直接路径**: 当前 `page=HOME`，在消息列表中的目标联系人列表项，应使用 `role=chat_target` 命中缓存并点击。
*   **搜索路径**: 如果列表页找不到，需点击顶栏的搜索入口，输入关键词，然后点击搜索结果中的第一个匹配项。
*   **判定成功**: 进入 `page=CHAT`，且屏幕顶部区域（约占 15%）的节点描述与目标联系人匹配，即成功。

### 2.2 文本输入模式切换与输入 (Text Sending)
底部输入栏的典型结构为：
`[ 模式切换图标 ] [ 中间输入区 ] [ 表情图标 ] [ 附件/更多图标 (+) ]`

**A. 判定输入模式：**
1. **文本模式 (Text Mode)**: 底部输入栏包含一个 `editable=true` 的 EditText 节点。
2. **非文本模式 (Non-Text Mode)**: `page=CHAT`，但 UI tree 中找不到任何 `editable=true` 的节点。

**B. 模式切换流程 (当处于非文本模式时)：**
1. **操作**: 必须点击底部输入栏最左侧的**模式切换图标**（通常是第一个 ImageView）。
2. **目的**: 强制切换输入模式，使下一次 UI dump 能够捕获到 `editable=true` 的 EditText 节点。
3. **输入**: 切换成功后，再执行 `input_text(role=message_input)`。

### 2.3 媒体发送流程 (Media Sending Flow)
媒体发送流程是多阶段的，需要严格遵循结构信号：

**步骤 1: 触发附件面板**
*   **入口**: 点击底部输入栏同一行里**最右侧**的附件/更多 `+` 图标（`role=compose_plus`）。
*   **禁止误点**: `+` 左边紧挨着的笑脸/表情图标不是附件入口。点它会打开表情面板。

**步骤 2: 进入相册选择器**
*   **结构判定**: 在附件面板中，寻找带有“图片/相册/照片”标签的格子，点击进入相册。
*   **角色**: 使用 `role=media_picker_entry`。

**步骤 3: 选择目标媒体**
*   **目标**: 在相册网格中，选择最新、最符合目标的媒体。
*   **识别原则**: 真实缩略图的格子内通常是**纯 ImageView 或 ImageView + 选择框/勾选覆盖层**，没有“扫描文档”等文字标签。
*   **优先选择**: 优先选择网格中第一个没有文字标签的纯缩略图或带勾选框的缩略图。
*   **角色**: 使用 `role=gallery_first_item`。

**步骤 4: 确认发送**
*   **状态**: 流程会进入一个 `page=UNKNOWN` 的预览/确认页。
*   **操作**: 必须点击右下角的**“发送/Send”**按钮，才能将图片发送到聊天页。
*   **角色**: 使用 `role=gallery_send_button`。

**步骤 5: 成功判定与等待**
*   **成功信号**: 媒体发送成功，最可靠的信号是 `page` 从 `UNKNOWN` (picker/预览页) 或 `CHAT` 切回 `CHAT`，且输入框内容被清空或恢复占位符。
*   **等待机制**: 如果点击 `gallery_send_button` 后，页面短时间仍停留在 `UNKNOWN` / 预览 / loading 状态，**不要按 `back`**。应等待 UI 刷新或使用 `wait_ui` 观察页面回到 `CHAT`。

## 3. 文本与媒体发送的通用总结

| 步骤 | 目标 | 结构信号/角色 | 备注 |
| :--- | :--- | :--- | :--- |
| **1. 导航** | 进入目标会话 | `role=chat_target` | 优先使用缓存的会话列表定位。 |
| **2. 模式检查** | 确保处于文本输入模式 | 检查是否存在 `editable=true` 节点 | 如果不存在，必须先点击底部输入栏最左侧的模式切换图标。 |
| **3. 输入文本** | 写入消息内容 | `role=message_input` | 写入目标文本。 |
| **4. 发送文本** | 发送文本消息 | `role=send_button` | 触发发送动作。 |
| **5. 发送媒体** | 发送图片/文件 | 遵循 2.3 的五步流程 | 必须通过 `compose_plus` -> `media_picker_entry` -> `gallery_first_item` -> `gallery_send_button`。 |

## 4. 成功判定的严格条件与恢复模式

### 4.1 成功判定（必须同时满足）
真正的发送成功，不仅是 `page=CHAT`，还必须满足以下三个条件：
1. **动作来源**: 上一步是 `tap_node` (不是 `back`、`swipe`、`home`)。
2. **页面转换**: 上一步的 `before_page` 是非 CHAT (picker / UNKNOWN / 预览页)。
3. **顶栏一致性**: 当前 CHAT 页的**顶栏文字**与媒体发送开始前你所在聊天的顶栏文字**一致**（确保没有误入其他会话）。

### 4.2 常见错点与恢复模式
| 现象 | 原因分析 | 推荐处理 |
| :--- | :--- | :--- |
| **误入表情面板** | 误点了输入栏左侧的笑脸/表情图标。 | 不要点表情或键盘图标，改点输入栏最右侧 `+`；必要时先 `back` 收起面板。 |
| **发送后页面卡死** | 媒体上传或发送处理中。 | **等待 UI 回到 CHAT**；不要 `back`，否则可能取消发送并跳到会话列表。 |
| **页面跳转到非预期状态** | 误触了表情、位置或附件等子面板。 | 使用 `back` 回退到上一级页面，再重新尝试目标节点。 |
| **文本输入框无法输入** | 处于非文本模式。 | 参照 2.2 流程，点击底部输入栏最左侧的模式切换图标。 |
| **发送失败/回退** | 确认发送后，页面状态异常。 | 检查是否满足 4.1 的三个成功条件。如果不满足，先 `back` 回会话列表，重新打开目标会话，并重新尝试发送。 |
