# 01 · 项目定位

## 1.1 一句话定位

**AI + 人工协同的企微私域运营智能体平台**：通过 Android RPA 模拟真人操作企业微信，配合 Web 客服工作台与后端 AI Agent，实现私域客户的自动接待、运营触达与长期记忆管理。

## 1.2 不是什么

为避免范围漂移，明确划定边界：

- ❌ 不是单纯的「企微自动回复机器人」。
- ❌ 不是「群发工具」。
- ❌ 不依赖企微官方 API（无客户消息接口资质），完全走客户端 RPA。
- ❌ 不做电商交易闭环，只做"运营触达 + 引导转化"。

## 1.3 核心闭环

```
企微客户消息
  → Android 监听并回调
  → 后端会话/AI 决策
  → 知识库检索（Milvus + Neo4j）
  → AI 生成 or 转人工
  → 风控校验
  → 后端设备队列 + ReAct 原语驱动 Android 执行
  → 回写消息与运营记录
```

## 1.4 关键用户角色

| 角色 | 在系统中的工作 |
| --- | --- |
| 运营管理员 | 配置策略、知识库、SOP、AI 人设、风控规则 |
| 人工客服 | 在 Web 工作台接管高价值/复杂会话，发文本/图/文件 |
| 设备管理员 | 管理 Android 机群、监控在线状态、处理异常 |
| 数据 / 业务负责人 | 看转化、回复率、AI 命中率、被删率 |

## 1.5 成功指标（北极星）

- **AI 自动回复占比** ≥ 60%（在不影响转化的前提下）。
- **AI 回复命中率（知识库 Top-K）** ≥ 80%。
- **客户 24h 内回复率** ≥ 95%。
- **被删率 / 投诉率** 维持在行业基线以下（具体阈值在 [风控](modules/risk.md) 中定义）。
