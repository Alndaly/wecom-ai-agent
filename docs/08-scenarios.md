# 08 · 业务场景

## 8.1 新客欢迎（MVP4）
触发：新客户被加好友 / 通过好友请求

```
add_friend.approved
  → 读 contact.source / channel
  → 选欢迎策略（按渠道、按时段）
  → AI 生成欢迎语（带 nickname）
  → 风控
  → send_text
  → 写入 user_memories
```

要求：
- 个性化（用昵称、用渠道措辞）
- 不模板化（同一策略 N 个变体，随机）
- 可配置语气（亲切 / 专业 / 俏皮）

## 8.2 场景识别
分类：
`咨询` / `报价` / `售后` / `投诉` / `高意向` / `流失风险` / `沉默唤醒` / `闲聊`

由 AI Workflow `intent_detect` 节点完成，结果写 `messages.intent` 与 `conversations.last_intent`。

## 8.3 老客促活（MVP4）
- 触发：`now - last_message_at > N days`（按客户阶段不同 N 不同）
- 生成：基于客户历史摘要 + 当前活动池
- 频次：同客户 30 天内最多 ≤ 2 次（风控强约束）

## 8.4 SOP 触达（MVP4）
- 用户在 `阶段 X` 停留 `> M 天` → 触发动作集合
- 动作可以是：发消息、改标签、提醒人工

## 8.5 朋友圈运营（MVP4）
- 选号 → 选内容池 → AI 文案 → 配图 → 定时
- 同一文案在多号发布需做"差异化改写"，否则风控会标记
