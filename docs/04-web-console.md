# 04 · Web 管理台

## 4.1 定位

不是配置后台，而是 **运营 + 客服 + 设备监控** 的合一工作台。

## 4.2 信息架构

```
登录
└── 工作台 (默认页)        ← 客服核心入口
    ├── 会话列表 (左)
    ├── 聊天窗口 (中)
    └── 用户画像 / AI 推荐 / KB 命中 / 历史摘要 (右)

侧边栏
├── 设备管理
├── 知识库
├── 用户/客户
├── 运营策略 (SOP / 朋友圈 / 欢迎语 / Prompt)
├── 数据分析
├── 团队与权限
└── 操作审计
```

## 4.3 客服工作台 — 关键交互

| 交互 | 说明 |
| --- | --- |
| 实时消息 | WebSocket 推送，未读数高亮，桌面通知（可选） |
| 模式切换 | 每会话支持「AI / 人工 / 混合」三态切换，人工模式下自动回复调度器跳过 |
| 快捷回复 | 团队共享 + 个人收藏；支持变量（如 `{{nickname}}`） |
| AI 推荐 | 右侧实时展示 AI 给出的候选回复（点击即发送） |
| KB 命中 | 展示当前消息检索到的 Top-3 知识片段，可一键溯源 |
| 会话转接 | 转给其他客服 / 转回 AI |
| 离开警告 | 有未读 / 未发草稿时关闭页面给提示 |

## 4.4 AI / 人工模式语义

- **AI**：用户消息直接进 AI Workflow，无须人工介入。
- **人工**：AI 不回复；消息只推 Web 工作台。
- **混合**：默认走 AI，命中以下任一规则即转人工——
  - AI 置信度 < 阈值（默认 0.55）
  - 命中关键词（投诉、退款、客诉等）
  - 用户处于高意向阶段
  - 用户主动要求人工

当前实现不使用 Redis 接管锁；Web 切到「人工」后，自动回复调度器不再为该会话触发 AI。人工发送会把本会话仍未反馈的客户入站消息标记为已进入回复队列，发送成功后统一记为已反馈。

## 4.6 验收标准（MVP1）

- [ ] 客服能登录、查看会话列表、打开会话、看到历史消息。
- [ ] 客户发来消息能在 ≤ 1s 内显示在工作台。
- [ ] 客服输入文本 → 发送 → 看到状态变为「发送中 → 已发送 / 失败」。
- [ ] 切换 AI / 人工模式立即生效。
